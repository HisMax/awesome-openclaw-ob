# Constitutional AI

## 一句话理解

> 传统方式训练AI安全，需要大量人类去标注"这个回答有害/无害"——又贵又慢。Constitutional AI的思路是：**给AI一份"宪法"（一组原则），让AI自己审查自己的回答**，就像给孩子一本家规，让他自己判断自己做得对不对。

## 核心方法

Constitutional AI（CAI）由 [[Anthropic 公司分析|Anthropic]] 提出，分为两个阶段：

### 阶段一：监督学习（SL-CAI）

1. 让模型生成回答
2. 模型根据"宪法原则"自我批评（如"这个回答是否鼓励暴力？"）
3. 模型根据批评修正自己的回答
4. 用修正后的回答微调原始模型

### 阶段二：强化学习（RL-CAI / RLAIF）

1. 让微调后的模型生成多个回答
2. 用另一个AI模型（而非人类）根据宪法原则评估哪个回答更好
3. 用这些AI偏好数据训练一个奖励模型
4. 用奖励模型做强化学习训练

## RLHF vs RLAIF

| 维度 | RLHF | RLAIF（CAI方式） |
|------|------|------------------|
| 偏好来源 | 人类标注员 | AI根据原则自动评估 |
| 成本 | 每条 ≥$1（甚至$10+） | 每条 <$0.01 |
| 可扩展性 | 受限于人力 | 几乎无限扩展 |
| 一致性 | 标注员间有分歧 | 原则驱动，更一致 |
| 透明度 | 偏好隐含在标注中 | 原则公开可审查 |

## 为什么CAI让Claude更"安全"

1. **原则可审查**：宪法内容是公开的，任何人都能审视 Anthropic 给 [[Claude 模型系列|Claude]] 设定的价值观
2. **减少人类偏见**：不依赖个别标注员的主观判断，而是依据系统化的原则
3. **链式推理**：AI在评估时会进行思维链推理（类似思维链提示技术），判断质量随模型能力提升而提升
4. **渐进式改善**：批评-修正过程可以反复应用，逐步降低有害性

## CAI 在 Agent 安全中的作用

在 AI Agent 场景中，CAI 训练出的模型天然更适合执行自主决策任务——因为其内置的价值观约束降低了 Agent 在[[Agent Execution Loop|执行循环]]中"失控"的概率。[[System Prompt 设计]]可以进一步强化 CAI 的行为边界。然而，CAI 并不能完全防范 [[Prompt Injection 风险|Prompt Injection]]，因为对抗性输入可能绕过宪法原则的判断。

## CAI的局限

- 宪法原则本身的制定仍然依赖人类价值判断
- AI评估器可能继承底层大语言模型的偏见
- "安全"和"有用"之间的平衡仍然是开放问题
