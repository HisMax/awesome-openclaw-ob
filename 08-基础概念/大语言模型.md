# 大语言模型

> **一句话本质：** 大语言模型就是一个读过互联网上几乎所有文字的"超级补全器"——你给它开头，它根据统计规律预测最合理的下一个词，反复预测就生成了完整回答。

## 核心原理

LLM 的能力建立在三根支柱之上：

1. **Transformer 架构**：2017年 Google 提出的"注意力即一切"架构，让模型能同时关注输入中任意位置的关系，而非像 RNN 那样逐词处理。这是 LLM 能理解长文本的根本原因。
2. **注意力机制（Attention）**：模型在生成每个词时，会给输入序列中的每个词分配一个"重要性权重"。就像你读一段话时，眼睛会自动聚焦到关键词上。
3. **预训练 + 微调范式**：先用海量无标注文本做自监督预训练（学会语言的通用规律），再用人类反馈做微调（RLHF/Constitutional AI），使其对齐人类意图。

## 参数规模演进

- **2020** GPT-3: 175B — 首次证明"大力出奇迹"
- **2023** GPT-4: 估计 1.8T (MoE) — 质的飞跃
- **2024** Llama 3: 405B — 开源追上闭源
- **2025-2026** MoE 成为主流：总参数动辄 600B+，但每次推理只激活 30-40B，兼顾能力与效率

关键趋势：参数规模不再是唯一竞争维度。2026年的进步更多来自数据质量、推理时计算（inference-time scaling）、工具调用和架构优化。这场竞争的全貌详见 [[2026 前沿模型竞争格局]]。

## LLM 在 Agent 框架中的角色

在 [[OpenClaw 是什么]] 等 Agent 框架中，LLM 扮演的是**"大脑"角色**：

- **感知层**：理解用户意图和环境上下文
- **决策层**：规划任务步骤、选择工具，驱动 [[Agent Execution Loop]]
- **生成层**：产出代码、文本、指令

LLM 本身不执行动作，它通过 [[Tool Use 机制]] 和 [[MCP 协议]] 调用外部工具来改变世界。这也是为什么 [[模型无关架构]] 如此重要——Agent 框架不应绑死在某一个 LLM 上。
