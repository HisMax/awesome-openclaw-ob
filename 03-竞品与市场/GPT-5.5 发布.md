# GPT-5.5 发布

## 一句话理解

> 2026-04-23 发布的 GPT-5.5 是 OpenAI 迄今最强模型——1M 上下文、$5/$30 定价、Terminal-Bench 82.7%，但在核心编码 benchmark SWE-Bench Pro 上以 58.6% 落后 [[Claude Opus 4.7-4.8 发布|Opus 4.8]] 的 69.2% 超过 10 分。5.5 Instant 于 5 月 5 日免费开放，引发模型民主化新一轮竞赛。

## 基本信息

| 维度 | 详情 |
|------|------|
| **发布日期** | 2026-04-23 |
| **内部代号** | Spud |

与 [[Claude Opus 4.7-4.8 发布|Opus 4.8]] 的定价对比：

| 维度 | GPT-5.5 | Opus 4.8 Standard | Opus 4.8 Fast |

输入价格相同，但 GPT-5.5 的输出价格贵 20%。考虑到 GPT-5.5 "72% 更少的输出 token 完成等效任务"的特性，实际成本差距可能被部分抵消。

### 如何解读这些数字

GPT-5.5 在自家 benchmark（Terminal-Bench、GDPval）上表现强劲，但在行业通用的 SWE-Bench Pro 上落后 Opus 4.8 超过 10 分。这是一个值得玩味的信号：

1. **SWE-Bench Pro 是最接近"真实软件工程"的测试**——使用活跃维护的真实代码库，没有 ground-truth 泄露风险
2. **10.6 分的差距是显著的**——不是统计误差范围内的差异，而是"能力等级的差距"
3. **但 GPT-5.5 在 token 效率上领先**——72% 更少的输出 token 意味着在高吞吐场景下 GPT-5.5 可能更经济

## GPT-5.5 Instant：模型民主化

### 时间线

| 日期 | 事件 |
|------|------|
| 2026-04-23 | GPT-5.5 API 发布，ChatGPT Plus/Pro 可用 |
| 2026-05-05 | **GPT-5.5 Instant 向所有免费用户开放** |

GPT-5.5 Instant 替代 GPT-5.3 Instant 成为 ChatGPT 所有计划（包括免费版）的默认模型。这意味着全球数亿免费用户一夜之间获得了接近最强前沿模型的能力。

### 意义

这是 OpenAI 的"Android 策略"——通过免费开放最强模型的轻量版，最大化用户基数和数据飞轮。相比之下，Anthropic 的免费版仍使用 Sonnet 而非 Opus，策略更保守。

对 AI 编码工具市场的影响：当免费用户也能用上接近旗舰级的模型时，"用更好的模型"作为付费卖点的说服力下降。工具层的编排、工作流、生态集成变得更重要。

## 与 Claude Opus 4.8 的全面对比

### 能力维度

| 维度 | GPT-5.5 | Opus 4.8 | 胜出 |
|------|---------|----------|------|
| SWE-Bench Pro | 58.6% | 69.2% | **Opus 4.8** |
| SWE-Bench Verified | — | 88.6% | **Opus 4.8** |
| Terminal-Bench 2.0 | 82.7% | — | **GPT-5.5** |
| Online-Mind2Web | — | 84% | **Opus 4.8** |
| Token 效率 | 72% 更少 | 基线 | **GPT-5.5** |
| 幻觉控制 | 显著改善 | 4x 诚实度提升 | 各有侧重 |

### 哲学分野

这不仅是技术差异，更是产品哲学的分野：

**OpenAI 的赌注**：原始 Agent 吞吐量 + token 效率。GPT-5.5 的设计目标是"在相同预算下完成更多任务"——更少的 token、更快的速度、更广的覆盖。

**Anthropic 的赌注**：诚实度 + 校准不确定性。Opus 4.8 的设计目标是"每个任务完成得更可靠"——主动报告代码缺陷、更精准的指令遵循、更强的安全对齐。

对于 AI 编码工具来说，这个分野意味着：
- 用 GPT-5.5 更适合高吞吐批量任务（CI/CD 流水线、批量代码迁移）
- 用 Opus 4.8 更适合高风险精细任务（生产环境修复、安全审计）

### 市场节奏

GPT-5.5 发布后 35 天，Anthropic 就推出了 Opus 4.8——这是 Opus 系列史上最快的迭代。这种"回合制竞赛"的节奏说明：

1. 两家公司都在密切监控对方的发布
2. 模型储备允许快速"应答式发布"
3. 企业用户是最终受益者——竞争驱动价格下降和能力提升

## GPT-5.5 在编码工具中的应用

### GitHub Copilot

GPT-5.5 成为 [[GitHub Copilot 2026年Q2更新|GitHub Copilot]] 的核心模型之一，但 Copilot 的计费模型变更（转向 usage-based billing）意味着使用 GPT-5.5 的成本更加透明也更加昂贵。

### Cursor

[[Cursor 2026年Q2更新|Cursor]] 在 Auto 模式中引入 GPT-5 路由，允许模型自动选择。但 Cursor 的多模型策略意味着 GPT-5.5 只是选项之一，而非唯一。

### 对 OpenClaw 的影响

1. **token 效率是竞争变量**：GPT-5.5 的 72% token 减少意味着 OpenClaw 如果做成本优化，GPT-5.5 在批量场景下可能比 Opus 更经济
2. **免费可用性改变竞争基线**：GPT-5.5 Instant 免费后，"模型能力"本身不再是壁垒，编排和工作流才是
3. **多模型混用策略价值提升**：不同模型各有所长，OpenClaw 如果能智能路由（精细任务用 Opus、批量任务用 GPT-5.5），比单一模型方案更有优势
