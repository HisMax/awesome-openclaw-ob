# GitHub Copilot 分析


![[assets/copilot.jpg]]

## 基本信息

- **类别**：AI 编码助手
- **定位**：IDE 内代码补全与生成
- **所属**：微软 / GitHub
- **底层模型**：GPT 模型系列

## 市场表现

- ~**42%** 市场份额（付费 AI 编码工具中最高，按用户数计）
- **2000 万+** 累计用户
- ~~130 万~~ → **470 万** 付费用户（2026.1，YoY +75%）
- 约 90% Fortune 100 公司部署
- 预估 ARR ~$4.5 亿

## 微软新动向：Copilot Tasks

微软推出 **Copilot Tasks**——"一个会自动执行的待办列表"，详见 [[OpenClaw vs Copilot Tasks]]。这标志着微软正式进入 AI Agent 赛道，从辅助工具走向自主执行。

## 最新动态（2026年3月）

- **四个专业化智能体**：GitHub 推出了四个专业化 Copilot Agent，分别聚焦代码审查、安全扫描、测试生成和文档维护。这与多Agent协作架构的行业趋势一致——从"一个通用 Agent"走向"多个专家 Agent 协作"
- **GPT-5 mini 集成**：Copilot 底层引入 OpenAI 的 GPT-5 mini 模型，在保持低延迟的同时显著提升了代码理解能力。Copilot 深度绑定 OpenAI 模型，与 Claude Code 绑定 Claude 的策略类似
- **日活用户 180 万**：月活和日活数据持续增长，日活跃用户达到 **180 万**，付费用户数增长至 **470 万**
- 面对 Cursor Automations 和 Devin 2.0 的竞争压力，Copilot 正在从"代码补全工具"向"自主编码平台"转型

## 最新动态（2026年Q2）

- **Usage-based billing（AI Credits）**：2026.6.1 起所有计划切换为按量计费，1 AI Credit = $0.01 USD。Pro $10/月含 $10 额度，Pro+ $39/月含 $39 额度。代码补全和 Next Edit Suggestions 仍无限制，其余功能按 token 消耗计费（详见 [[GitHub Copilot 2026年Q2更新]]）
- **开发者反弹**：新计费模式引发大量不满——有用户报告数小时内耗尽整月额度，社区出现"逃离 Copilot"声浪
- **Copilot App 技术预览**：桌面 App 带来 canvases、语音对话、cloud sessions 和 agentic browsing——从 IDE 插件向 Agent 原生桌面应用形态进化
