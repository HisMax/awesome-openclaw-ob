# MCP 协议 2026年3月进展

2026 年 3 月，[[MCP 2026 路线图|MCP 协议]] 迎来了关键的生态扩展——ChatGPT、VS Code（Copilot）、Cursor 三大平台同时宣布支持 MCP，加上 MCP Apps 交互式应用、Elicitation 结构化输入和 HTTP Hooks 的落地，MCP 正在从"Anthropic 主导的协议"变为"行业标准"。

## 生态扩展：三大平台同时支持

| 平台 | MCP 支持状态 | 说明 |
|------|-------------|------|
| **ChatGPT** | 已支持 | OpenAI 正式将 MCP 纳入 ChatGPT 的工具调用体系 |
| **VS Code (Copilot)** | 已支持 | [[GitHub Copilot 分析\|GitHub Copilot]] 通过 VS Code 原生支持 MCP 服务器 |
| **[[Cursor 分析\|Cursor]]** | 已支持（v2.6+） | 通过 MCP Apps 提供交互式 UI 支持 |
| **Claude Code** | 原生支持 | 作为 MCP 的发起者，Claude Code 一直深度支持 |

### 为什么这是里程碑

当 OpenAI（ChatGPT）和 Microsoft（VS Code/Copilot）同时采纳一个由 Anthropic 主导的协议时，说明 MCP 已经跨过了"厂商锁定"的信任门槛，成为了真正的行业标准。这类似于当年 Google 和 Microsoft 同时支持 HTTP/2 的意义——协议的价值在于普遍采纳。

## MCP Apps：交互式应用

在 [[MCP 2026 路线图]] 的 UI 扩展方向上取得实质进展：

- **Cursor v2.6 率先落地**（参见 [[Cursor 2026年3月更新]]）：MCP 服务器可以在 Cursor 中渲染富交互界面
- Agent 不再只能返回文本，可以直接生成图表、表单、可视化数据面板
- 这打开了"Agent as App"的新范式——MCP 服务器本身就可以是一个应用

## MCP Elicitation：结构化输入请求

- **Claude Code 率先支持**（参见 [[Claude Code 2026年3月更新]]）：MCP 服务器可以向 Agent 请求结构化输入
- 场景示例：Agent 在执行数据库迁移时，MCP 服务器弹出结构化表单让用户确认目标数据库、迁移策略、回滚方案
- 本质上是 MCP 从"Agent → 工具"单向通信进化为"双向交互"——工具也可以"反问"Agent

## HTTP Hooks 支持

- MCP 服务器现在支持 HTTP Hooks 作为传输层
- 这与 [[Claude Code v2.1 更新日志|Claude Code v2.1.68]] 引入的 HTTP hooks 形成协议级对齐
- HTTP Hooks 让 MCP 可以与 CI/CD 流水线、监控系统、企业内部 API 等传统基础设施无缝对接
- 对比 [[MCP 2026 路线图]] 中的传输可扩展性方向，HTTP Hooks 是继 stdio 和 SSE 之后的第三种传输方式

## 行业意义

MCP 在 3 月的进展让 [[MCP 2026 路线图]] 中的愿景加速成为现实。当所有主要 AI 编码工具都支持同一个协议时，工具生态的互操作性将大幅提升——一个为 Claude Code 开发的 MCP 服务器，可以直接在 Cursor 和 Copilot 中使用。这对 [[ClawHub 官方技能注册表]] 等技能市场有深远影响：MCP 的标准化让"一次开发，处处运行"成为可能。

## Q2 更新（2026年4-6月）

### MCP 2026-07-28 Release Candidate

MCP 规范发布了重大 Release Candidate（2026-07-28），核心变化：

- **无状态协议内核**：远程 MCP 服务器不再需要粘性会话或共享会话存储，可直接运行在普通轮询负载均衡器之后
- **Extensions 框架**：正式引入扩展机制
- **Tasks**：任务管理能力纳入规范
- **Authorization 硬化**：OAuth 2.1 基线映射完成
- **正式弃用策略**：首次引入 deprecation policy

### 传输层改进

Streamable HTTP 传输新增 `Mcp-Method` 和 `Mcp-Name` 头用于路由；列表和资源读取结果新增 `ttlMs` 和 `cacheScope` 参数（模仿 HTTP Cache-Control），客户端可精确知道 tools/list 响应的有效期。

### 生态规模

截至 2026 年 5 月 24 日，MCP 官方 Registry API 记录了 **9,652 个最新服务器**和 **28,959 个 server/version 记录**。SDK 月下载量超过 **9,700 万次**（Python + TypeScript）。

### AAIF 新托管项目

2026 年 6 月 4 日，[[Agentic AI Foundation（AAIF）]] 接纳 **agentgateway** 作为最新托管项目——一个专为 MCP、A2A、LLM 和 API 流量设计的开源网关。

### 与路线图的对照（更新）

|-----------|---------|------|
| 传输可扩展性 | 无状态协议内核 + HTTP 路由头 | RC 阶段 |
| 智能体通信 | Elicitation + Tasks | 已纳入 RC |
| 治理成熟化 | 弃用策略 + OAuth 2.1 | 初步落地 |
| 企业就绪 | Server Cards + 缓存控制 | 显著推进 |
| MCP Apps | Extensions 框架 | RC 阶段 |
