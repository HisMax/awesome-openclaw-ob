# Cognition Series D 融资

> **一句话总结**：2026 年 5 月 27 日，Cognition（Devin 的创造者）完成 $10 亿 Series D 融资，估值从 8 个月前的 $102 亿跃升至 $260 亿。最惊人的不是融资数字，而是 Devin 现在编写了 Cognition 自身 89% 的代码——从 2025 年 12 月的 13% 到 89%，AI 编码 Agent 正在用自己来证明自己。

## 核心数据

| 指标 | 数值 | 对比 Series C（2025.9） |
|------|------|----------------------|
| **融资金额** | $10 亿+ | $4 亿（+150%） |
| **估值（post-money）** | $260 亿 | $102 亿（+155%） |
| **年化营收（ARR）** | $4.92 亿 | $0.37 亿（+1,230%） |
| **企业使用量** | >10x 增长 | 年初基数 |
| **Devin 自编写代码比例** | 89% | 13%（2025.12） |

## 营收增长：从 $37M 到 $492M

Cognition 的收入增长曲线极为陡峭：

| 时间 | ARR | 说明 |
|------|-----|------|
| 2025.5 | $3,700 万 | Devin 商业化初期 |
| 2025.9（Series C） | — | $102 亿估值 |
| 2026.5（Series D） | **$4.92 亿** | 12 个月增长 13 倍 |

12 个月内 ARR 增长 13 倍（$37M → $492M），月均增长率约 50%。这种增速在 SaaS 历史上极为罕见，说明企业对 AI 编码 Agent 的需求已从"实验"转为"刚需"。

## 89% 代码由 Devin 编写

这是本次融资最引人注目的数据点：

| 时间 | Devin 自编写代码比例 |
|------|---------------------|
| 2025.12 | 13% |
| 2026.5 | **89%** |

从 13% 到 89%，6 个月增长了近 7 倍。这意味着 Cognition 的工程团队中，人类工程师的角色已从"写代码"转变为"审核和指导 Devin 写代码"——与 [[案例-Reorx 从程序员到 Tech Lead|Reorx 的"程序员到 Tech Lead"转变]] 高度吻合。

### 自指式验证（Self-Referential Validation）

Devin 编写自己的代码是一种强有力的产品验证（dogfooding）：
- **如果 Devin 不好用**，Cognition 的工程师就不会让它写 89% 的代码
- **如果 Devin 引入 bug**，Cognition 自己会最先受到影响
- 这种"AI 用自己来改进自己"的循环，是 Agentic AI 最令人振奋（也最令人不安）的趋势之一

## Windsurf 收购与品牌统一

### 收购背景

2025 年 7 月，Cognition 收购了 Windsurf（前 Codeium）的剩余资产：

| 维度 | 详情 |
|------|------|
| **被收购方** | Windsurf（编码工具创业公司） |
| **竞购方** | OpenAI、Google、Cognition |
| **Google 的交易** | $24 亿收购 Windsurf 核心工程团队和授权 |
| **Cognition 获得** | 技术、客户、选择留下的员工 |

### 品牌统一：Windsurf → Devin Desktop

收购后，Cognition 将 Windsurf 更名为 **Devin Desktop**，实现品牌统一：

| 产品 | 说明 |
|------|------|
| **Devin（Cloud）** | 云端自主编码 Agent |
| **Devin Desktop**（原 Windsurf） | 本地 IDE 编码体验 |
| **Devin Local** | 本地运行的 Agent |

### ACP 集成

Devin Desktop 原生支持 [[ACP v1 生态扩展|Agent Client Protocol（ACP）]]，这意味着 Devin 可以在任何 ACP 兼容编辑器（JetBrains、Zed、Neovim、Emacs）中运行。Cognition 的策略很清晰：不再试图构建自己的编辑器，而是成为最好的 Agent，让编辑器选择留给用户。

## SWE-1.6 模型

Cognition 同步推出了自研的 SWE-1.6 模型：

| 指标 | 数值 |
|------|------|
| 推理速度 | 最高 950 tokens/秒 |
| 定位 | 专为软件工程任务优化 |

SWE-1.6 的高速推理能力是 Devin 能实现"企业使用量 10x 增长"的技术基础之一——更快的推理意味着更短的 Agent 执行时间，直接影响用户体验和企业生产力。

## Mercedes-Benz 案例深度

Mercedes-Benz 的使用案例值得单独展开：

| 维度 | 详情 |
|------|------|
| **任务** | COBOL 遗留代码分析与现代化 |
| **代码量** | 200,000+ 行 COBOL |
| **传统预估工期** | 8 个月 |
| **Devin 实际工期** | 8 天 |
| **加速比** | ~30x |

COBOL 现代化是全球企业 IT 的痛点——银行、保险、制造业中仍有大量 COBOL 系统在运行。如果 Devin 能系统性地解决 COBOL → 现代语言的迁移问题，这本身就是一个巨大的市场。

## 关键洞察

### 1. AI 编码 Agent 的"iPhone 时刻"

从 $37M 到 $492M 的 12 个月增长，说明 AI 编码 Agent 已从"早期采用者的玩具"变成"企业的生产工具"。就像 iPhone 不是第一部智能手机但定义了智能手机市场一样，Devin 可能不是第一个 AI 编码 Agent 但正在定义这个市场。

### 2. 89% 的天花板在哪

如果 Devin 编写 89% 的代码，剩下的 11% 是什么？大概率是：架构设计、需求澄清、安全审计、跨系统集成决策——这些恰好是人类工程师最有价值的工作。89% → 100% 的最后一步可能是最难的，也是人类角色最终会稳定的位置。

### 3. 军方客户的信号

U.S. Army 和 Navy 成为 Devin 客户，说明 AI 编码 Agent 已通过了军方级别的安全审查。这对整个 AI 编码行业的企业采纳都是重大利好。

### 4. 估值隐忧

$260 亿估值 / $4.92 亿 ARR = 52.8x 倍数。这个倍数远高于 [[Anthropic Series H 融资|Anthropic 的 20.5x]]，说明投资人对 Cognition 的增速预期更激进。如果增速放缓，估值压力会更大。
