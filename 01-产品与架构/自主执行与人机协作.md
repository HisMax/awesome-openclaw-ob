## 自主执行的定义和边界

费曼式理解：**自主执行就是"你告诉 Agent 目的地，它自己决定怎么开过去"。**

但"自己开"不等于"闭着眼开"。自主执行有明确的边界：
- Agent 在**定义好的范围内**自由行动
- 遇到**超出能力或授权**的情况，停下来问人
- 所有操作**有日志可审计**

2026 年行业共识：企业需要的不是完全自主的 AI，而是**可问责的 AI**——能独立思考，但有条件地行动。

## 两种人机协作模式

### Human-in-the-Loop（人在环中）
Agent 每做一步关键操作前都要问你：
> "我准备删除这个文件，可以吗？"

**适用场景**：高风险操作、刚开始使用 Agent 时、涉及金钱/隐私的任务。

### Human-on-the-Loop（人在环上）
Agent 自己跑，你在旁边监控仪表盘，只在出问题时介入：
> Agent 默默处理了 100 封邮件，你只在它准备回复老板时收到提醒。

**适用场景**：已建立信任后、低风险重复任务、效率优先的场景。

## 信任和安全的平衡：95/5 模型

2026 年的最佳实践叫 **"95/5 模型"**：

- **95%** 的任务让 Agent 自主处理
- **5%** 的边缘情况或高风险操作交给人决策

这不是偷懒，而是精明——把人类的注意力集中在真正需要判断力的地方。

## OpenClaw 的权限等级设计

OpenClaw 把这个光谱变成了实际可操作的权限系统：

| 权限等级 | 行为 | 例子 |
|---------|------|------|
| **只读** | Agent 只能看，不能改 | 查询天气、读取文件 |
| **建议** | Agent 提出方案，等你点确认 | "建议重命名这10个文件" |
| **受限执行** | 在沙箱内自由操作 | 在隔离环境中运行代码 |
| **完全执行** | 直接操作，事后通知 | 自动回复低优先级邮件 |

用户可以**按 Skill 粒度**设置权限——网购 Skill 需要每次确认，文件整理 Skill 可以全自动。

## 何时让 Agent 自主 vs 何时要人确认

一个简单的判断框架：

```
可逆的操作？ ——是→ 可以自主
  ↓否
涉及金钱/隐私？ ——是→ 必须人确认
  ↓否
有明确规则？ ——是→ 可以自主（设好规则）
  ↓否
需要判断力？ ——是→ 人确认
```

> "Human-in-the-loop 不是临时方案——它是构建可信 AI Agent 的长期模式。"

## Auto Mode：第三条路（v2026.6）

v2026.6 引入的 **Auto Mode** 是自主执行与人机协作理念的工程落地——在 Ask（每条命令都人工审批）和 YOLO（跳过所有审批）之间找到了第三条路。Auto Mode 采用三层审批架构：策略匹配（白名单直接执行）→ 模型审核（独立审核模型判断风险）→ 人工兜底（不确定项交给人类）。审核模型与 Agent 模型分离，且只能允许单次低风险执行，不能给出永久授权。详见 [[OpenClaw v2026.6 版本更新]]。
