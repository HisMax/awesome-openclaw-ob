# Agent 默认超时策略

## 一句话理解

> v2026.3.22 将 Agent 默认超时从 600 秒提升到 **48 小时**——相当于把员工的"午休时间限制"改成了"两天出差额度"，标志着 OpenClaw 从"对话工具"向"长时运行自主代理"的范式跃迁。

## 变更内容

| 属性 | 旧值 | 新值 |
|------|------|------|
| 默认超时 | 600 秒（10 分钟） | 172,800 秒（48 小时） |
| 适用范围 | 所有 Agent 会话 | 所有 Agent/ACP 会话 |
| 配置方式 | 固定 | 可按 Agent 覆盖 |

## 为什么是 48 小时

600 秒的限制来自早期 OpenClaw 作为"聊天助手"的定位——用户发问、Agent 回答，整个交互在几分钟内完成。但随着 [[Agent Execution Loop]] 支持越来越复杂的任务链（代码生成 → 测试 → 修复 → 部署），10 分钟已远远不够。

48 小时的选择反映了两个趋势：
1. **ACP（Agent Controlled Process）** 的成熟——Agent 现在可以自主规划和执行多步骤任务，参见 [[自主执行与人机协作]]
2. **Vibe Coding 场景**——开发者启动一个 Agent 后去做其他事，期望 Agent 自主完成复杂工程任务，详见 [[Vibe Coding]]

## 安全考量

超时延长也带来风险：一个失控的 Agent 在 48 小时内可以消耗大量计算资源和 API 调用。这使得 [[权限控制机制]] 和 [[沙箱运行时注入防护]] 变得更加关键。
