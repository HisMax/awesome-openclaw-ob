# 自主决策循环

自主决策循环是 AI Agent 区别于普通 LLM 对话的核心特征。Agent 不仅生成文本回复，还能**自主规划步骤、调用工具、评估结果并决定下一步行动**，形成一个持续的决策-执行-反馈循环。

## 核心机制

在 [[Agent Execution Loop]] 的 Phase 5（Tool Execution Loop）中，自主决策循环表现为：

1. 模型分析任务需求
2. 决定调用哪个工具
3. 执行工具并获取结果
4. 评估结果是否满足需求
5. 如果不满足，**自动决定下一步行动**（调用其他工具或调整策略）
6. **重复直到产出最终文本响应**

这个循环的关键在于：模型自主判断何时继续、何时终止，无需人工干预每个步骤。

## 人在回路的控制

OpenClaw 的设计哲学是"人在回路，交互协作"（human-in-the-loop），用户可以随时介入：

- **`steer` 模式**：在 tool boundary（工具调用边界）优雅抢占，让 Agent 转向新指令
- **`interrupt` 模式**：立即中断当前执行
- **`followUp` 模式**：排队等待当前循环完成后再执行

这与 AutoGPT 的"设定目标，放手不管"（fire-and-forget）形成鲜明对比。参见 竞品对比总览。

## 与 AutoGPT 的对比

| 维度 | AutoGPT | OpenClaw |
|------|---------|----------|
| 控制模型 | Agent 自主决定所有步骤 | 用户随时可以 steer / interrupt |
| 循环行为 | 常见无限循环、重复动作 | Lane Queue 保证确定性终止 |
| 交互模式 | fire-and-forget | human-in-the-loop |

> "The difference isn't intelligence — it's architecture."

## 自我修改能力

OpenClaw 的自主决策循环还延伸到了代码层面——Agent 可以修改自己的源代码（参见 Tool Use 机制），这使得决策循环不仅影响运行时行为，还能改变 Agent 本身的能力边界。

## 后续演进

v2026.4.29 引入 **Active-Run Steering** 默认模式——用户在 Agent 运行中发送的后续消息默认作为方向调整注入活跃运行，而非排队等候，减少消息丢失。v2026.6 的 **Auto Mode** 则在执行审批层面为自主决策循环加入了策略→模型→人工三级安全网。详见 [[OpenClaw v2026.4 版本更新]] 和 [[OpenClaw v2026.6 版本更新]]。
