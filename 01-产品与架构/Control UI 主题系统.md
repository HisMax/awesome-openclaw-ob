# Control UI 主题系统

## 一句话理解

> 主题系统就像给房子选装修风格——Roundness 滑块决定"家具的边角有多圆"，Knot 主题则是一套"黑红工业风"的整体方案，而 WCAG 2.1 AA 合规确保色盲用户也能"看清每一件家具"。

## Roundness 滑块（v2026.3.22）

v2026.3.22 在 [[Dashboard 控制面板]] 的设置中引入了 **Roundness 滑块**，允许用户统一调节整个控制界面的边框圆角半径（border-radius）。这不是一个纯粹的美学选择——圆角大小直接影响 UI 元素的视觉层级和可点击区域感知。滑块范围从 0（完全直角，适合信息密度高的专业场景）到最大值（圆润风格，适合移动端触控场景）。

所有 UI 组件共享同一个 Roundness 变量，确保按钮、卡片、输入框、弹窗等元素的圆角风格一致。这是 v2026.3.22 "统一主题边框圆角" 工作的核心成果。

## Knot 主题精修（v2026.3.23）

v2026.3.23 热修复版本对 **Knot 主题** 进行了精修，将配色方案收敛为黑红双色调。更关键的是，这次精修通过了 **WCAG 2.1 AA 对比度标准**——所有文本与背景的对比度至少达到 4.5:1（普通文本）和 3:1（大字体），确保低视力用户和色觉异常用户也能正常使用。

同一版本还统一了按钮原语组件，将分散的按钮样式整合为一套遵循 [[WCAG 无障碍标准]] 的基础组件。这意味着未来所有新增的 UI 元素都会自动继承无障碍合规属性。

## 贡献者

**@BunsDev** 是主题系统这两次迭代的主要贡献者，负责 Roundness 滑块的实现、Knot 主题配色精修以及 WCAG 合规验证。这体现了 OpenClaw 社区中 UI/UX 专项贡献者的角色——不是所有贡献都是后端架构，前端体验同样是开源项目竞争力的关键。

## 与整体 UI 架构的关系

主题系统运行在 [[Dashboard 控制面板]] 的前端层面，受益于 v2026.3.12 的模块化视图拆分——主题变量可以按模块粒度覆盖。系统感知的深色主题（v2026.3.22 在 Android 端引入）与 Roundness 滑块互补：前者控制"明暗"，后者控制"方圆"，共同构成用户可自定义的视觉体验。
