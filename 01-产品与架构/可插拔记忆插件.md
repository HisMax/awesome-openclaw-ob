# 可插拔记忆插件

## 一句话理解

> 把 Agent 的"长期记忆"从硬编码的内置模块变成了可更换的"记忆芯片"——你可以给 Agent 装一个 LanceDB 向量记忆、一个 SQLite 结构记忆、或者任何自定义后端，它们都通过统一接口注入系统提示词。

## 架构

v2026.3.22 引入可插拔记忆插件系统，核心变更：

- `memory_search` 和 `memory_get` 工具**独立注册**，不再互相抑制（此前是一个已知 Bug）
- 记忆插件通过 [[Plugin 扩展系统]] 的 `openclaw.extensions` 机制注册
- 插件可向系统提示词（System Prompt）注入记忆相关上下文
- Memory-LanceDB 插件首次使用时自动引导到插件运行时

## 与记忆系统的关系

这是 [[记忆系统]] 的架构升级。此前 OpenClaw 的记忆能力是内置的、不可替换的。可插拔化意味着：

1. **社区可以贡献记忆后端**——比如用 Pinecone 做云端记忆、用 Redis 做高速缓存记忆
2. **企业可以对接内部知识库**——将企业文档系统作为 Agent 的"长期记忆"
3. **研究者可以实验新方法**——不同的记忆检索算法可以作为插件切换测试

## 技术要点

- 记忆插件注册时声明 `owner`（防止 [[Marketplace 安装范围验证|范围扩展攻击]]）
- 上下文引擎在 `assemble()` 阶段整合记忆插件注入的内容，参见 [[上下文引擎 modelId 适配]]
- 紧凑目录回退确保记忆内容不会因 Token 预算超限而被丢弃
