# /btw 旁问命令

## 一句话理解

> `/btw` 就像在手术进行中悄悄问旁边的护士一个问题——**不打断手术流程**（不改变会话上下文），也**不动用手术器械**（不调用工具），只是快速获得一个答案。

## 工作原理

传统聊天中，每条消息都会修改会话上下文——Agent 记住你说过的话，后续回答基于整个对话历史。`/btw` 打破了这个模式：

1. 用户输入 `/btw 什么是 SSRF？`
2. Agent 生成一个**无工具回答**——纯文本，不调用任何 [[Tool Use 机制|Tool]]
3. 回答**不写入会话上下文**——之后的对话仍基于 `/btw` 之前的状态
4. 等效于一个"临时窗口"

## 使用场景

- 在复杂任务执行中快速查询概念定义
- 确认一个参数值而不打断 Agent 的任务流
- 请求格式转换或翻译而不污染上下文

## 设计哲学

`/btw` 体现了 [[Agent Execution Loop]] 中"上下文是宝贵资源"的理念。随着 [[上下文引擎 modelId 适配|上下文引擎]] 对 Token 预算的精细管理（紧凑目录回退机制），保持上下文的纯净变得越来越重要。`/btw` 给用户提供了一个"安全旁路"。
