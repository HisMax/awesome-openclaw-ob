# 捆绑提供商插件化迁移

## 一句话理解

> v2026.3.22 将 OpenRouter、GitHub Copilot 和 OpenAI Codex 从"硬编码在核心"迁移为"独立插件"——就像把单体应用的内置功能拆分成微服务，每个提供商现在可以独立更新、独立启用、独立认证。

## 迁移的提供商

| 提供商 | 定位 | 插件化后变化 |
|----------|------|----------------|
| **OpenRouter** | 模型聚合平台 | 独立插件 + 动态模型元数据获取 |
| **GitHub Copilot** | 代码补全 | 独立插件 + 动态 model ID 前向兼容 |
| **OpenAI Codex** | 代码生成 | 独立插件，默认 `openai-codex/gpt-5.4` |

## 为什么迁移

迁移前，这些提供商的代码深嵌在 OpenClaw 核心中——更新一个提供商的 API 需要修改核心代码并重新发布整个框架。迁移到 [[Provider-Plugin 架构]] 后：

1. **独立更新**：提供商可以在不升级 OpenClaw 核心的情况下更新
2. **独立认证**：插件管理自己的 OAuth/API-key 流程
3. **可选加载**：不使用某个提供商的用户无需加载其代码，优化 [[Gateway 编译插件加载|启动性能]]

这与 v2026.3.12 中 Ollama、vLLM、SGLang 的插件化是同一浪潮的延续，说明 OpenClaw 正在系统地将所有提供商从核心中解耦。
