# NanoClaw 与 Docker 合作

> **一句话总结**：[[NanoClaw]] 创始人 Gavriel Cohen 从 Hacker News 发布到与 Docker 达成合作仅用 6 周，展示了 OpenClaw 生态中轻量化替代方案的快速崛起。

## 合作详情

- **日期**：2026 年 3 月 13 日宣布
- **创始人**：Gavriel Cohen
- **从发布到合作**：仅 6 周（Hacker News 首发 → Docker 合作）

## NanoClaw 回顾

[[NanoClaw]] 是一个轻量级 AI Agent 框架，核心引擎仅约 4,000 行代码，默认以容器化方式运行。[[Andrej Karpathy]] 在对 [[OpenClaw 是什么|OpenClaw]] 表达安全顾虑时曾推荐它作为替代品。

## 与 Docker 合作的意义

1. **容器化天然契合**：NanoClaw 默认容器化运行的设计理念与 Docker 的核心业务高度一致
2. **安全叙事的商业化**：NanoClaw 的"最小攻击面"理念正在从开源理念转化为商业产品
3. **生态加速**：Docker 的分发渠道将大幅降低 NanoClaw 的使用门槛
4. **6 周速度**：从 HN 发布到企业合作的速度，再次印证了 [[OpenClaw 生态系统形成速度|OpenClaw 生态的形成速度]]

## 对 OpenClaw 生态的影响

NanoClaw 与 Docker 的合作并非对 OpenClaw 的威胁，而是生态多样化的体现。正如 [[OpenClaw 核心生态项目]] 所示，多层次的解决方案（从轻量级的 NanoClaw 到全功能的 OpenClaw，再到企业级的 [[NVIDIA NemoClaw 企业版|NemoClaw]]）构成了完整的 Agent 框架光谱。

## Q2 更新

NanoClaw 在合作公布后快速增长：

- GitHub Stars 突破 **20,000**
- 累计下载量超过 **100,000 次**
- 最新版本 **v1.2.53**（2026 年 4 月 15 日），增强了 token 计数和 SDK 配置
- 每个 NanoClaw Agent 运行在独立的 MicroVM 级 Docker Sandbox 中，实现操作系统级隔离
