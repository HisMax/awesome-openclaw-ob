# 社区与开源生态


![[assets/open-source-eco.jpg]]

## 一句话理解

OpenClaw 的社区像一场自发的"数字龙虾派对"——两个月内从零到 237K GitHub Stars、118K Discord 成员、866 贡献者、5,705 个社区技能。这种增长速度在开源史上前所未有，比 Kubernetes 快 18 倍。

## GitHub 增长传奇

GitHub Stars 增长轨迹打破了历史纪录：

| 时间节点 | Stars |
|----------|-------|
| 发布第 1 天 | 5,000 |
| 第 3 天 | 60,000+ |
| 2026.1.26 单日 | +25,310（GitHub 历史纪录） |
| 2026.2.25 | 226,887 |

84 天从 9K 到 200K，仅 React、Python、Linux、Vue 在其之上。

### Stars 争议

但这份数据并非没有争议。多次单日 25,000+ 的跳跃被独立 GitHub Archive 数据标记为异常，AI CERTs 分析了可能的机器人刷星行为。项目的实际使用量和生态规模确实庞大，但增长数据可能被夸大。

## 开源治理

### MIT 许可证

OpenClaw 使用 MIT 许可——最宽松的开源许可之一，允许任何人自由使用、修改、商业化。

### 基金会转型

OpenClaw Foundation 的建立是社区治理的关键转折：

- Peter Steinberger 加入 OpenAI 后，项目转交独立基金会
- OpenAI 赞助基金会运营
- 维护者委员会正在组建中
- 社区 5 天内自发捐款 $35,000

但治理仍面临挑战——与 Linux Foundation 这样的成熟模式相比，缺乏技术顾问委员会、独立性保障和安全响应团队。

## 核心洞察

OpenClaw 的社区增长证明了一个规律：**当技术足够简单到让普通人使用时，社区就会爆炸式增长**。WhatsApp 作为交互界面、MIT 开源许可、模型无关——这三个选择让 OpenClaw 成为了"人人都能用的 AI Agent"，而非"只有程序员才能玩的技术玩具"。

但快速增长也带来了 [[安全边界与风险（总览）|安全问题]] 和 质疑声——增长速度远超安全治理能力，这是整个 Agentic AI 行业的缩影。
