# 飞书集成

## 一句话理解

> [[OpenClaw v2026.3 版本更新|v2026.3.22]] 引入深度飞书（Feishu/Lark）集成，使 OpenClaw 能在字节跳动的企业协作平台上实现结构化交互、审批流程和子代理会话绑定。

## 背景

飞书（国际版 Lark）是字节跳动旗下的企业协作平台，在中国科技企业中拥有广泛用户基础。OpenClaw 此前已支持 WhatsApp、Telegram、Discord、Slack 等 20+ 消息平台，飞书的深度集成标志着 OpenClaw 在中国企业市场的进一步渗透。

## 核心能力

### 结构化交互

- **交互式审批卡片**：在飞书中发送结构化卡片，支持一键审批/拒绝操作
- **快捷操作启动器**：通过飞书卡片按钮快速触发 OpenClaw Agent 动作
- **身份感知卡片头部和注释页脚**：卡片根据用户身份动态展示不同信息

### 会话与上下文

- **回调用户和会话上下文保持**：飞书回调事件中正确保持用户身份和会话上下文
- **当前会话 ACP 与子代理会话绑定**：在飞书对话中发起的 Agent Controlled Process 自动绑定到当前会话
- **显式线程回复**：支持飞书的消息线程功能

### 思考过程展示

- **`onReasoningStream` 和 `onReasoningEnd`**：支持将 LLM 的思考 Token 以 Markdown 格式实时展示在飞书消息中
- 用户可以看到 Agent 的推理过程，增强透明度和信任

### 扩展动作

- 消息已读/编辑
- 显式线程回复
- 消息置顶
- 成员检查

## 安全

- 飞书签名 Webhook 验证使用**常量时间比较**，防止时序攻击
- 详见 [[2026年3月安全公告汇总]]

## 意义

飞书集成的深度程度（结构化卡片、ACP 绑定、思考过程展示）远超一般的消息平台适配，更接近"原生应用级"集成。这与 [[中国用户与 OpenClaw]] 的市场热潮和 [[OpenClaw GitHub 数据更新 2026Q1|Tencent/Volcengine 赞助]] 共同表明，OpenClaw 在中国企业市场的定位正从"开发者工具"转向"企业协作基础设施"。
