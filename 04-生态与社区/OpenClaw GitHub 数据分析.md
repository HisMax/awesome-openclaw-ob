# OpenClaw GitHub 数据分析

## 增长轨迹

OpenClaw 在 GitHub 上创造了惊人的增长记录：

| 时间节点 | Stars |
|----------|-------|
| 发布第 1 天 | 5,000 |
| 第 3 天 | 60,000+ |
| 2026.1.26 单日 | **+25,310**（GitHub 历史纪录） |
| 2026.2.2 | 140,000 |
| 2026.2.14 | ~175,000 |
| 2026.2.25 | **226,887** |
| 2026.2.28 | **~237,000** |
| 2026.3（Q1） | **334,000+**（详见 [[OpenClaw GitHub 数据更新 2026Q1]]） |
| 2026.4 初 | **350,600+**（超越 React，GitHub 史上最多 Stars 软件项目） |
| 2026.5-6（Q2 最新） | **375,000+**（详见 [[OpenClaw GitHub 数据更新 2026Q2]]） |

## 历史对比

- 84 天从 9K → 200K，仅 React、Python、Linux、Vue 在其之上
- 比 **Kubernetes 快 18 倍**
- GitHub 全站排名第 15，软件项目排名第 5
- 项目使用 TypeScript 编写，发布在 npm 上

## Stars 争议

部分观察者认为增长存在异常：

- 多次单日 25,000+ 的跳跃被独立 GitHub Archive 数据标记为异常
- AI CERTs 的报道分析了可能的机器人刷星行为
- 但项目本身的实际使用量和生态系统规模也确实庞大
- [[Gary Marcus 对 OpenClaw 的批评|批评者]] 将此作为 OpenClaw "过度炒作"的证据之一

这一争议也在 [[OpenClaw Hacker News 社区讨论]] 中被多人提及。
