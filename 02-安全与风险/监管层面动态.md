# 监管层面动态

各国监管机构已开始关注 [[OpenClaw 是什么|OpenClaw]] 及 AI Agent 带来的安全和隐私挑战。

## 各机构动作

### 荷兰数据保护局（AP）

- 警告用户对 GDPR 合规负责
- 呼吁明确 AI Agent 是否在《AI 法案》范围内
- 强调 AI Agent 处理个人数据时，用户承担数据控制者责任

### NIST

- 启动 **"AI Agent 标准计划"**
- 目标：确保 Agent 安全地代表用户行动
- 关注互操作性和安全标准制定

### 比利时 CCB

- 发布关于 [[ClawJacked 远程代码执行漏洞|CVE-2026-25253]] 的紧急安全警告
- 要求相关组织立即更新

### 新加坡 CSA（2026年5月28日）

- 发布专项安全警告（AD-2026-005），明确针对 OpenClaw 等自主 AI Agent 的安全风险
- 个人建议：避免在含敏感数据的设备上运行、最小权限账户、仅安装可信来源 Skill、要求人工审批高风险操作
- 企业建议：**零信任架构**、缩小 Agent 权限范围、定期轮换专用凭证、部署策略执行代理、持久化日志、人工审批不可逆操作
- 这是东南亚首个针对 AI Agent 的国家级安全警告

### 阿曼 FSA（2026年5月25日）

- 识别三个关键安全缺陷，发布针对金融机构的强制安全指导

## 监管趋势

AI Agent 领域的监管从 Q1 的早期阶段进入 Q2 的**实质性行动阶段**：

1. 现有数据保护法规（如 GDPR）将适用于 Agent 处理的数据
2. **国家级安全机构直接介入**（新加坡 CSA、阿曼 FSA）——从行业警告上升到政府指导
3. AI 专项法案可能将 Agent 纳入监管范围
4. 安全标准正在制定中（NIST AI Agent 标准计划）
5. [[GTIG AI 生成零日攻击报告]] 中国家级攻击者利用 OpenClaw 的证据可能加速监管立法
