# 权限控制机制


![[assets/permission-control.jpg]]

[[OpenClaw 是什么|OpenClaw]] 的权限控制是其安全模型的核心组成部分，但存在多处设计缺陷。

## 信任模型

- 每个网关**一个受信操作者**，不支持多对抗性用户共享
- DM 配对：未知发送者收到配对码，需人工批准
- TOFU（Trust On First Use）策略（CVE-2026-25253 修复后引入）
- 信任模型的设计直接影响自主执行与人机协作的安全边界

## 关键权限作用域

| 作用域 | 功能 | 风险 |
|--------|------|------|
| `operator.admin` | 管理员权限 | 完全控制网关 |
| `operator.approvals` | 审批权限 | 可关闭所有确认提示 |
| `exec.approvals.set` | 执行审批开关 | 设为 off 则静默执行 |
| `tools.exec.host` | 执行环境 | 设为 gateway 则逃逸容器 |

## 已知绕过方式

1. **[[ClawJacked 远程代码执行漏洞]]**：通过窃取令牌获取完整管理员权限
2. **工具策略绕过**：结合 TOCTOU 竞态条件（约 25% 暴力成功率）
3. **localhost 错误信任**：本地连接被自动视为安全，实际上可被远程利用
4. **[[WebSocket 提权漏洞 GHSA-rqpp]]**：共享密钥连接自行声明 `operator.admin`（CVSS 10.0）
5. **[[配对机制安全缺陷]]**：bootstrap 令牌重放提权 + 配对载荷泄露永久凭证
6. **[[工作区插件自动加载 RCE]]**：本地插件自动加载绕过所有确认机制
7. **[[Claw Chain 四漏洞链]]**：MCP 回环非 Owner 提权（CVE-2026-44118），伪造 `senderIsOwner` 获取 owner 控制
8. **CVE-2026-32922**（CVSS 9.9）：`rotateDeviceToken` 函数缺少 scope 验证，配对 token 一步变管理员

## 核心设计问题

> "任何能向 Agent 发消息的人，实际上被授予了与 Agent 本身相同的权限。" —— [[致命三合一安全矛盾]]

这导致 Prompt Injection 可以直接利用 Agent 的全部权限。恶意 Skills 和数据泄露因此变得更加严重——攻击者无需突破权限系统，只需通过提示注入即可获取与 Agent 等同的访问能力。
