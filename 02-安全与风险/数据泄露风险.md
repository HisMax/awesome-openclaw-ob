# 数据泄露风险

[[OpenClaw 是什么|OpenClaw]] 作为个人 AI Agent，拥有对用户私有数据的广泛访问权限，使数据泄露成为高概率风险。

## 数据泄露途径

1. **[[凭证泄露与信息窃取]]**：Infostealer 窃取配置文件中的明文密钥和 Agent 记忆
2. **[[Prompt Injection 风险]]**：恶意指令诱导 Agent 外泄敏感信息
3. **[[恶意 Skills 供应链攻击]]**：恶意 Skill 静默外泄数据
4. **[[大规模实例暴露]]**：未认证的公网实例直接暴露数据
5. **Moltbook 数据库泄露事件**：第三方平台配置错误暴露 AI Agent 数据

## Agent 数据的特殊价值

AI 个人助手是被入侵机器上**最有价值的单一目标**，因为它集中存储了：

- SSO 会话和凭证
- 文件系统访问记录
- 通信历史
- 日常活动和日程蓝图
- 私人消息内容

比攻击者数月横向移动获取的信息更加丰富和集中。记忆系统中存储的用户偏好和行为模式更进一步放大了数据泄露的影响。

## 防御层面

沙箱机制和权限控制机制是减轻数据泄露风险的关键手段。

## GDPR 合规问题

荷兰数据保护局（AP）已警告用户需对 GDPR 合规负责，涉及 AI Agent 处理的个人数据。

## Q2 更新

[[Claw Chain 四漏洞链]] 中的 TOCTOU 读取逃逸（CVE-2026-44113）和 Heredoc 环境变量泄露（CVE-2026-44115）为数据泄露提供了即使在沙箱内也能利用的新路径。[[GTIG AI 生成零日攻击报告]] 确认国家级行为者已在利用 OpenClaw 进行数据发现与外泄。
