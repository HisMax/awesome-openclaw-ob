# 安全厂商评估汇总

各大安全厂商对 [[OpenClaw 是什么|OpenClaw]] 的安全评估普遍负面，印证了 Agentic AI 在安全领域面临的根本性挑战。

## 厂商评价

| 厂商 | 核心评价 |
|------|---------|
| **Microsoft** | 应视为具有持久凭证的**不受信代码执行**。不适合在标准工作站上运行。如必须评估，应部署在**完全隔离环境** |
| **Cisco** | 安全是**可选的，不是内建的**。实测发现 ClawHub 最热门 Skill 包含静默数据外泄 |
| **CrowdStrike** | 提示注入将成为他们的 **PrintNightmare 时刻** |
| **Trend Micro** | 不受限的可配置性使 OpenClaw **不适合日常使用** |
| **Kaspersky** | 使用**专用备用计算机或 VPS**。不要在主要计算机上安装 |
| **Aikido.dev** | 尝试保护 OpenClaw 是**荒谬的**。全面强化基本上把它变成了带额外编排的 ChatGPT。**它只在危险时才有用** |
| **Palo Alto Networks** | **2026年最大的潜在内部威胁** |
| **McAfee** | OpenClaw 对个人用户存在**显著安全风险**，不建议在含敏感数据设备上使用 |

## 2026 年 Q2 新增政府/监管机构评估

| 机构 | 核心评价 | 时间 |
|------|---------|------|
| **新加坡 CSA** | 发布专项安全警告，建议企业采用零信任架构、人工审批不可逆操作、定期轮换凭证 | 2026-05-28 |
| **阿曼 FSA** | 识别三个关键安全缺陷，发布强制安全指导 | 2026-05-25 |
| **Cyera** | 披露 [[Claw Chain 四漏洞链]]，证实沙箱可被 TOCTOU 竞态绕过 | 2026-05-18 |
| **GTIG (Google)** | 确认国家级行为者将 OpenClaw 作为攻击基础设施，[[GTIG AI 生成零日攻击报告|首次确认 AI 生成零日]] | 2026-05-11 |

## 共识

所有厂商均认为 OpenClaw 不适合在生产环境或主要工作设备上运行。核心分歧仅在于是"完全不用"还是"在严格隔离环境中使用"。这些评估也与 [[凭证泄露与信息窃取]] 和 [[恶意 Skills 供应链攻击]] 的实际案例相互印证。2026 年 Q2 政府机构的介入（新加坡 CSA、阿曼 FSA）标志着 OpenClaw 安全问题从行业关注上升到**国家级监管议题**。
