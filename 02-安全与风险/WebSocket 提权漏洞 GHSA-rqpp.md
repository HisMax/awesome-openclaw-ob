# WebSocket 提权漏洞 GHSA-rqpp

**一句话总结**：共享密钥认证的 WebSocket 连接可自行声明 `operator.admin` 权限，CVSS 评分 **10.0 满分**，是 OpenClaw 迄今最严重的单一漏洞。

## 漏洞基本信息

| 项目 | 详情 |
|------|------|
| GHSA 编号 | GHSA-rqpp-rjj8-7wv8 |
| CVSS 评分 | **10.0**（Critical） |
| 影响版本 | <=2026.3.11 |
| 修复版本 | 2026.3.12 |
| 漏洞类型 | 认证提权 |
| 攻击前提 | 拥有共享密钥的认证连接 |

## 漏洞分析

### 根因

OpenClaw 的 WebSocket 认证机制中，通过共享密钥（shared secret）建立的连接在声明权限作用域时**缺乏服务端验证**。攻击者只需拥有共享密钥即可在连接中自行声明 `operator.admin` 权限作用域，而服务端不会校验该声明是否合法。

### 与 CVE-2026-25253 的关系

此漏洞与此前的 [[CVE-2026-25253 ClawJacked 漏洞详解]] 形成呼应——两者都利用了 WebSocket 层面的信任缺陷：

- **CVE-2026-25253**：通过跨站 WebSocket 劫持（CSWSH）绕过 localhost 信任获取访问
- **GHSA-rqpp**：在已认证连接上通过自行声明提升到最高权限

两者共同揭示了 OpenClaw WebSocket 认证体系的**系统性设计缺陷**。

### 攻击路径

1. 攻击者获取共享密钥（配置文件明文存储，参见 [[凭证泄露与信息窃取]]）
2. 使用共享密钥建立合法 WebSocket 连接
3. 在连接中声明 `operator.admin` 权限作用域
4. 服务端接受声明，攻击者获得完整管理员权限
5. 可执行任意操作：关闭审批、逃逸容器、执行命令

### 影响

CVSS 10.0 意味着该漏洞在所有维度上都达到了最高评级：

- **攻击向量**：网络可达
- **攻击复杂度**：低
- **权限要求**：低（仅需共享密钥）
- **用户交互**：无需
- **影响范围**：机密性、完整性、可用性全部受损

## 权限作用域问题

该漏洞直接涉及 [[权限控制机制]] 中记录的关键权限作用域：

| 作用域 | 攻击者可达成 |
|--------|-------------|
| `operator.admin` | 自行声明即获得 |
| `operator.approvals` | 可关闭所有审批确认 |
| `exec.approvals.set` | 可设为 off 实现静默执行 |
| `tools.exec.host` | 可设为 gateway 逃逸容器 |

一旦获得 `operator.admin`，攻击者拥有与 [[CVE-2026-25253 ClawJacked 漏洞详解|ClawJacked 漏洞]] kill chain 最后阶段相同的能力。

## 修复措施

2026.3.12 版本修复了此漏洞，核心改进：

- 服务端对权限作用域声明进行**强制验证**
- 共享密钥连接不再允许自行声明高权限作用域
- 权限声明与认证方式绑定——不同认证方式对应不同的最大权限上限

## 安全建议

1. **立即升级**至 2026.3.12 或更高版本
2. **轮换所有共享密钥**——旧密钥可能已被利用
3. 审计网关日志，检查是否有异常的 `operator.admin` 权限声明
4. 参考 [[安全最佳实践]] 配置最小权限
