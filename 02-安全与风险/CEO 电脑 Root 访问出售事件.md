# CEO 电脑 Root 访问出售事件

**一句话总结**：2026年2月22日，BreachForums 用户以 $25,000 出售一家英国公司 CEO 电脑的 root 访问权，其中 OpenClaw 作为集中式情报枢纽包含的数据比传统横向移动数月获取的信息还要丰富。

## 事件详情

**时间**：2026年2月22日

BreachForums 用户 **"fluffyduck"**（2026年2月新注册，仅 10 帖，0 声誉）发布出售帖，目标是一家**英国工业自动化公司** CEO 的电脑 root 访问权。

| 项目 | 详情 |
|------|------|
| 要价 | **$25,000**（接受 Monero / Litecoin） |
| 卖家 | fluffyduck（2026年2月注册，10帖，0声誉） |
| 目标 | 英国工业自动化公司 CEO |
| 出售内容 | root shell + 公司数据库 + 网站后端 + AI 助手完整访问 |

## 出售内容详细清单

- **CEO 电脑 root shell**——完全控制权
- **完整公司数据库**（BSON 格式）
- **网站后端访问**
- **OpenClaw 完整访问**：所有对话、上下文、记忆、Dashboard 凭证
- **个人信息**：Trading 212 API 密钥（CEO 正在构建交易机器人）、家庭成员信息、联系方式

## 技术分析：OpenClaw 作为集中式情报枢纽

CEO 在工作电脑安装 [[OpenClaw 安全风险|OpenClaw]] 无意间创建了一个**集中式情报枢纽**：

- **SSO 会话**——所有企业系统的单点登录
- **凭证存储**——各类 API 密钥和密码
- **文件系统访问**——公司文档和代码
- **网络访问**——内部网络连通性
- **通信历史**——邮件、消息记录
- **AI 记忆文件**——用户的工作模式和决策逻辑

这种信息密集度使一台安装了 OpenClaw 的 CEO 电脑比传统攻击者**数月横向移动**获取的信息还要丰富。这违反了 [[安全最佳实践]] 中"不要在主要工作设备上运行"的基本原则。

> AI 个人助手正在成为被入侵机器上**最有价值的单一目标**。

## 与 Infostealer 趋势的关联

该事件与 [[Infostealer 窃取 AI Agent 灵魂]] 和 [[Hudson Rock 威胁情报报告]] 构成完整的威胁图谱：

- Infostealer 自动化窃取配置文件 → 暗网市场出售完整访问权 → 买家获得"AI 灵魂"级别的情报

这正是 [[暗网讨论分析|Flare 分析]] 中确认的攻击模式。

## 教训与洞察

- **最小特权原则**：CEO 等高价值目标不应在主工作电脑运行全权限 AI Agent
- **隔离部署**：如必须使用，应在独立的 [[沙箱机制|沙箱环境]] 中运行
- **AI Agent 加速信息集中**：传统攻击需要横向移动收集情报，AI Agent 主动帮攻击者完成了这一步
- **低门槛卖家的高价值数据**：0 声誉的新账户也能出售关键资产，说明获取门槛在降低
- 报道来源：Cato CTRL 和 SpyCloud 独立确认了该事件
