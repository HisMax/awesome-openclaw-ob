# 2026年3月安全公告汇总

2026年3月共披露 **10 个 GHSA 安全公告**，涵盖 1 个严重、4 个高危、5 个中危漏洞。集中暴露了 [[OpenClaw 安全风险|OpenClaw]] 在 WebSocket 认证、配对机制、插件加载、权限控制及多平台集成方面的系统性安全缺陷。

## 严重度分布

- **严重（Critical）**：1 个 — CVSS 10.0 满分的 WebSocket 提权
- **高危（High）**：4 个 — 涉及 DoS、提权、RCE、越权访问
- **中危（Moderate）**：5 个 — 涉及信息泄露、路径注入、授权绕过

## 漏洞分类分析

### 认证与权限缺陷
- GHSA-rqpp：共享密钥连接可自行声明 `operator.admin`，与 [[权限控制机制]] 中已知的权限作用域问题一脉相承
- GHSA-63f5：bootstrap 令牌可反复重放提升权限
- GHSA-r7vr：命令授权用户越权访问 Owner-only 接口
- GHSA-7h7g：配对载荷嵌入永久凭证，与 [[凭证泄露与信息窃取]] 中的明文凭证问题同类

### 供应链与代码执行
- GHSA-99qw：`.openclaw/extensions/` 下的插件自动加载无需用户确认，是 [[恶意 Skills 供应链攻击]] 在本地工作区的延伸

### 平台集成安全
- GHSA-jq3f：Telegram Webhook 验证前即读取并缓冲请求体，造成资源耗尽
- GHSA-xwcj：Telegram 媒体获取错误信息中泄露 Bot Token
- GHSA-g2f6：iMessage SCP 操作数中未验证 shell 元字符
- GHSA-f5mf：Zalo 白名单使用可变群组名称而非稳定 ID

### 文件权限
- GHSA-vr7j：JSONL 会话记录文件权限未设为 `0o600`

## 其他安全改进（同期）

除上述 GHSA 公告外，2026年3月还包含以下安全修复：

- **SQL LIKE 通配符注入修复** — 防止通过 LIKE 查询的通配符进行信息探测
- **Ruby 审批流程代码加载标志默认拒绝** — 将默认行为从允许改为拒绝
- **Unicode 混淆攻击防护** — 命令审批提示中防止 Unicode 字符混淆
- **exec approval 欺骗修复** — 防止绕过执行审批流程
- **网络 SSRF 风险修复** — 限制服务端请求伪造攻击面

### v2026.3.22 新增安全加固

[[OpenClaw v2026.3 版本更新|v2026.3.22]] 带来了全面的安全加固，覆盖以下领域：

**运行时注入防护**
- 沙箱环境阻止 JVM 注入（`MAVEN_OPTS`/`SBT_OPTS`/`GRADLE_OPTS`/`ANT_OPTS`）
- 阻止 Glibc 可调参数利用（`GLIBC_TUNABLES`）和 .NET 依赖劫持（`DOTNET_ADDITIONAL_DEPS`）
- `jq` 从默认安全 bin 白名单移除（`jq -n env` 信息泄露路径关闭）

**网络安全加固**
- 远程主机 `file://` 媒体 URL 和 UNC/网络路径在本地文件系统解析前被阻止
- 显式代理 SSRF 固定 + HTTPS 代理隧道翻译
- Windows SMB 凭证握手防护
- 语音通话 Webhook 预认证请求体限制 64 KB / 超时 5 秒

**认证与配对加固**
- iOS 设置码绑定到预期节点配置文件
- 首次使用引导兑换拒绝更广角色/范围请求
- 设备审批对照调用者会话范围检查
- 无设备身份的控制 UI 会话阻止特权配对审批 RPC

**执行白名单加固**
- macOS 白名单解析加固，防止 wrapper 和 `env` 欺骗
- `time` 在白名单评估中视为透明包装器
- 韩文空白填充码点在审批提示中转义
- 宿主环境变量覆盖处理在 Gateway 和 Node 间统一加固

**插件与 Marketplace 安全**
- 远程 Marketplace 清单条目验证安装范围扩展
- 外部 git/GitHub 源、HTTP 归档和绝对路径被拒绝
- 插件拥有的上下文引擎注册强制 owner 感知
- 核心 `legacy` 引擎 ID 欺骗防护

**消息平台安全**
- 飞书签名 Webhook 验证使用常量时间比较
- Nostr DM 解密前强制执行入站 DM 策略 + 速率/大小守卫
- Synology Chat 回复绑定稳定数字 `user_id`
- 邮件 Webhook 元数据外部内容包装前消毒
- Tlon DM 授权在引用展开前强制执行

## 趋势观察

3月的安全公告反映了几个显著趋势：

1. **攻击面扩大**：从核心 WebSocket 到 Telegram、iMessage、Zalo 等多平台集成，攻击面随生态扩张而急剧扩大
2. **配对机制脆弱**：多个漏洞（#3、#10）指向配对/引导流程的设计缺陷，详见 [[配对机制安全缺陷]]
3. **权限模型持续受压**：`operator.admin` 提权路径仍是高价值攻击目标，印证了 [[致命三合一安全矛盾]]
4. **供应链风险本地化**：GHSA-99qw 表明供应链攻击已从 ClawHub 延伸到本地工作区
5. **v2026.3.22 的安全回应**：该版本引入了迄今最全面的安全加固，从运行时注入防护到 Marketplace 安全验证，覆盖了从底层沙箱到上层插件生态的全栈安全
