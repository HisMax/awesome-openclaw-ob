# 代码执行安全

[[OpenClaw 是什么|OpenClaw]] 中代码执行是核心安全问题之一。Agent 具备在用户系统上执行任意命令的能力，一旦被攻击者控制，后果极其严重。这是 Agentic AI 架构中 自主执行能力带来的根本性挑战。

## 代码执行风险来源

1. **[[ClawJacked 远程代码执行漏洞]]**：攻击者通过 CSWSH 窃取令牌后可执行任意命令
2. **[[Prompt Injection 风险]]**：恶意指令可诱骗 Agent 执行系统命令
3. **恶意 Skills 供应链攻击**：恶意 Skill 在受信自动化上下文中执行
4. **权限提升**：攻击者可通过 API 关闭用户确认、强制命令在宿主机执行，绕过 [[权限控制机制]]
5. **[[工作区插件自动加载 RCE]]**：`.openclaw/extensions/` 下的插件自动加载无需用户确认
6. **[[WebSocket 提权漏洞 GHSA-rqpp]]**：共享密钥认证提权后可执行任意命令（CVSS 10.0）
7. **[[Claw Chain 四漏洞链]]**：TOCTOU 沙箱逃逸 + MCP 回环提权，实现宿主机完整 RCE（CVSS 最高 9.6）
8. **CVE-2026-32922**：配对 token 一步变管理员 + RCE（CVSS 9.9），被 ARMO 称为"OpenClaw 历史上最严重漏洞"

## 关键攻击面

- `tools.exec.host` 设置为 `gateway` 时，命令直接在宿主机执行，逃出 Docker 容器
- `exec.approvals.set` 设置为 `off` 时，所有命令执行无需用户确认
- **约 12,812 个暴露实例**可被直接 RCE 攻击利用

## Microsoft 评价

> "应视为具有持久凭证的**不受信代码执行**。不适合在标准工作站上运行。如必须评估，应部署在**完全隔离环境**。"

## 防护措施

- 启用 [[沙箱机制|Docker 沙箱]]（默认关闭，需手动开启）
- 保持用户确认机制开启
- 使用 最小权限原则
